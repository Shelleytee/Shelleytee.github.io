JavaScript vs HTML & CSS

In one of my awesome earlier blogs I discussed the place of HTML & CSS. What they are and how they fit in with the "web development" infrastructure. To read more about that [click here]...

In comparasion Javascript is the muscles and tendons that add movement, flow, action, and substance to webpages. Without Javascript, HTML & CSS are trendy looking grannies. We love them alot, but they are just so slow and boring.

Control Flow and Loops
The control flow is the order in which the computer executes statements in a script.

MDN - Mozilla Developer Network

Huh, you say? Well think of it like this. You wake up every morning, tumble out of bed, stumble to the kitchen, pour yourself a cup of ambition, yawn and stretch and try to come to life. Jump into the shower and the blood starts pumpin, out on the street the traffic starts jumpin. Now suppose each of those actions were a script, the computer ensures that everything is done in the right order. This is called the control flow.

A loop is a sequence of instructions that is continually repeated until a certain condition is met...

MDN - Mozilla Developer Network

Huh, you say again? Well think of it like this. You wake up every morning... So, what does that entail? Waking up every morning, the same way, regardless of where you are, what your wearing, or how your sleeping... Lets see how that would look as a loop. 

var alarm = true;
var snoring = true;
var dreaming = true;

while (alarm) {
  if (snoring || dreaming) {
    console.log("Keep sleeping");
  } else {
    console.log("What the fudge!");
    alarm=false;
    console.log("Moan...");
  }
}
Accessing Data via Array[] vs Object{}
An array is an ordered collection of data (either primitive or object depending upon the language). Arrays are used to store multiple values in a single variable.

MDN - Mozilla Developer Network

Huh, you say? You really need to stop saying huh. It's getting a little annoying. It is after all, 5am in the morning. So let's use the example of a single pouch toilet bag and what we may have in it. 

var myToiletBag = ["toothbrush", "toothpaste", "soap", ["purse", "coins", 3]];

So now you want to look at the toothpaste in your toilet bag. How will that look in code. 

console.log(myToiletBag[1]);

When we look something up in an array we will use the location number of where an item is located.

Object refers to a data structure containing data and instructions for working with the data.

MDN - Mozilla Developer Network

Huh, you say? I know, that's a curly one. But basically it's a lookup table. So now think of our earlier example of the toilet bag. A toilet bag has compartments. You may put your toothbrush and tooth paste in one compartment, your shampoo and hair brush in another compartment. And your electric razor in another compartment. So how would that look in code. 

var myToiletBag = {
  "compartment1": {
    "item1": "toothbrush",
    "item2": "toothpaste"
  },
  "compartment2": {
    "item1": "shampoo",
    "item2": "hairbrush"
  },
  "compartment3": {
    "item1": "electric razor"
  }
};

So now you want to look at the hair brush in your toilet bag. How will that look in code. 

console.log(myToiletBag.compartment2.item2);

When we look something up in an object we will use the names (eg compartment2.item2) of where a value (eg hair brush) is located.

Functions
A function is a code snippet that can be called by other code or by itself, or a variable that refers to the function. When a function is called, arguments are passed to the function as input, and the function can optionally return an output. A function in JavaScript is also an object.

MDN - Mozilla Developer Network

Huh, you say? Stop doing that. No more huh's. Basically a function is a task or calculation that does something that can be repeated each time you want to use it. So, drawing on our first example of: You wake up every morning, tumble outta of bed, stumble to the kitchen, pour yourself a cup of ambition, yawn and stretch and try to come to life. Jump into the shower and the blood starts pumpin. We call all of these tasks together "getting ready for work". So lets see how the first task would fit within a function. 

function gettingReadyForWork() {
  var alarm = true;
  var snoring = true;
  var dreaming = true;

  while (alarm) {
    if (snoring || dreaming) {
      console.log("Keep sleeping");
    } else {
      console.log("What the fudge!");
      alarm=false;
      console.log("Moan...");
    };
}

We could add each task we need to do to get ready for work into the function. This ensures the tasks are completed in the same order and in the same way each time the function is run. This gaurantees that even if you are still asleep while you are getting ready for work you will not forget anything. Our brains would simply call the function to start it, like below... 

gettingReadyForWork();
